ActiveMQ Artemis 2.42.0 Release Notes

  • A list of commits.

  • Please see the “Versions” chapter in the User Guide for upgrade instructions.

Note: This release requires use of Java 17 or above.

Bugs Fixed

  • ARTEMIS-3110 - PullMessage timeout is ignored
  • ARTEMIS-4140 - HTTP Transport - Wrong authority compoment in HTTP request causes 400 Bad Request
  • ARTEMIS-4967 - store.commit(txDeleteCursorOnReload); may not be reachable in processReload() method
  • ARTEMIS-4993 - ActiveMQIOErrorException for transformed messages
  • ARTEMIS-5134 - Incorrect address size calculated due to a race condition in AMQP application properties decode process
  • ARTEMIS-5349 - Bridges with concurrency can leak
  • ARTEMIS-5377 - Deadlock and Crash of Artemis Broker When Restarting a Bridged Peer
  • ARTEMIS-5444 - Refactor temp-queue-namespace
  • ARTEMIS-5446 - Memory leak on Artemis backup node with failover & failback
  • ARTEMIS-5452 - Hierarchical Topics extra size added
  • ARTEMIS-5458 - Incorrect capitalization of transport constant “connectionTTL” in documentation
  • ARTEMIS-5463 - ServerStatus.instance.server leaking instances between restarts
  • ARTEMIS-5465 - Connection router doesn’t resolve the USER_NAME key with TextFileCertificateLoginModule
  • ARTEMIS-5466 - ClassCastException in AuthenticationFilter after upgrade from 2.38.0
  • ARTEMIS-5467 - High CPU usage in OutboundStore.generateMqttId

  • ARTEMIS-5522 - MQTT LWT not sent after subsequent disconnects on same session

  • ARTEMIS-5540 - Expire MQTT 3.x session immediately when defaultMqttSessionExpiryInterval is 0

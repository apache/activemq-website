ActiveMQ Artemis 2.41.0 Release Notes

  • A list of commits.

  • Please see the “Versions” chapter in the User Guide for upgrade instructions.

Note: This release requires use of Java 17 or above.

Bugs Fixed

  • ARTEMIS-3351 - Explicitly disconnect consumers when closing session administratively
  • ARTEMIS-4777 - Broker connections (AMQP) with receiver operation not working
  • ARTEMIS-5253 - ActiveMQRAConnectionFactoryImpl fails to implement getReference()
  • ARTEMIS-5254 - MQTTSession.stop failure: Parameters not enabled.
  • ARTEMIS-5325 - Don’t block session creation/closing with sending management notification
  • ARTEMIS-5337 - Consumer + auto-create receives from existing wildcard queue
  • ARTEMIS-5358 - Log when core bridge fails to send a message
  • ARTEMIS-5372 - parsing invalid json properties should trap error and report in status
  • ARTEMIS-5373 - json properties need to support keySurround and only auto surround when not already surrounded
  • ARTEMIS-5379 - Journal commit record recorded before Page Writer is finished leading to possible message loss on a crash
  • ARTEMIS-5380 - json properties need to respect insertion order
  • ARTEMIS-5381 - auto increment of link name in amqp address federation can lead to missing messages on restart
  • ARTEMIS-5382 - Temporary network issues can cause persistent cluster subgroups
  • ARTEMIS-5384 - PagedTimeWriter should be stopped before a server shutdown
  • ARTEMIS-5385 - Paging check completion should be done at the page’s executor
  • ARTEMIS-5428 - Hardening Paging Operation around Server Shutdown
  • ARTEMIS-5439 - Value ‘ROLE_NAME’ is not facet-valid with respect to enumeration ‘[CLIENT_ID, SNI_HOST, SOURCE_IP, USER_NAME]’
  • ARTEMIS-5441 - Replication start will deadlock / starve if PageTimedWriter is flow controlled

New Features

  • ARTEMIS-5371 - Support HTTP/2 from the embedded web server
  • ARTEMIS-5437 - Add a more advanced message bridging feature to AMQP broker connections

Improvements

  • ARTEMIS-5207 - Expose persisted status of queue’s paused attribute
  • ARTEMIS-5351 - Harmonize AddressControl and AddressView
  • ARTEMIS-5360 - Syntax issues in Docker files
  • ARTEMIS-5368 - AMQP Federation prevent consumer loops for included federation consumers
  • ARTEMIS-5375 - provide a fileAlder32 checksum in status of property configuration
  • ARTEMIS-5388 - Expose core bridge connection status via JMX
  • ARTEMIS-5422 - AMQP Federation example needs hardening
  • ARTEMIS-5434 - Use full exception name in exceptions

Tasks

  • ARTEMIS-5056 - make auto-delete queues/addresses ‘default’ documentation clearer
  • ARTEMIS-5363 - Clean up old war references in docs & configs
  • ARTEMIS-5383 - Performance issues with HierarchicalRepositoryChangeListener
  • ARTEMIS-5387 - Test suite improvements
  • ARTEMIS-5426 - Update jar exclusions for new console

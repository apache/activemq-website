ActiveMQ Artemis 2.43.0 Release Notes

  • A list of commits.

  • Please see the “Versions” chapter in the User Guide for upgrade instructions.

Note: This release requires use of Java 17 or above.

Bugs Fixed

  • ARTEMIS-4924 - Handle invalid messages in SNF queues
  • ARTEMIS-5099 - Probable impossible cast in AMQPLargeMessage
  • ARTEMIS-5530 - Some handling of compressed messages can throw NegativeArraySizeException
  • ARTEMIS-5562 - NullPointerException on call closeConnectionFactory method in ActiveMQResourceAdapter class
  • ARTEMIS-5579 - Address not available while paging cleanup being performed
  • ARTEMIS-5581 - Discovery group deadlock on ServerLocator.close
  • ARTEMIS-5587 - HttpServerAuthenticator needs to set correct TCCL
  • ARTEMIS-5596 - New client w/old broker chokes on null ConfigurationManaged flag
  • ARTEMIS-5597 - Initial distribution broken when sending to FQQN
  • ARTEMIS-5598 - Mitigate race condition on ServerConsumer callback
  • ARTEMIS-5601 - Auto create queues in AMQP is not synchronizing bindings storage
  • ARTEMIS-5605 - AMQP Bridge and Federation receivers can NPE if closed before remote Attach arrives
  • ARTEMIS-5612 - Unable to disable queue metrics
  • ARTEMIS-5616 - Compressed Large Messages handled as regular messages are returning too many credits upon rollback
  • ARTEMIS-5634 - Transactions should be used on the server’s side when mirror source is in use
  • ARTEMIS-5639 - Fix MQTTRetainMessageManagerTest
  • ARTEMIS-5651 - AMQP Bridge policy managers missing some needed synchronized tags
  • ARTEMIS-5672 - Fix AddressManager Leaks
  • ARTEMIS-5675 - Server’s JVM won’t halt when storage is disconnected
  • ARTEMIS-5679 - The CLI perf command doesn’t close the connection factory
  • ARTEMIS-5683 - Properly convert wildcards in LegacyLDAPSecuritySettingPlugin
  • ARTEMIS-5687 - Closing an AMQP session via console or management does not send an End frame
  • ARTEMIS-5688 - Set routing-type when creating MQTT subscription queue
  • ARTEMIS-5689 - Tunneling Core Large Messages over broker connections can fail in some cases
  • ARTEMIS-5691 - Race between MQTT CONNACK & disconnection for auth failures
  • ARTEMIS-5693 - jgroups scaledown race with shared channel
  • ARTEMIS-5694 - Lingering sessions after a storage timeout
  • ARTEMIS-5696 - Tunneled core large messages can drop some bytes if flow controled at specific stages
  • ARTEMIS-5697 - Fix looping call from AMQP Sasl handling on no available mechanisms
  • ARTEMIS-5700 - KubernetesLoginModule doesn’t control HttpClient instances

New Features

  • ARTEMIS-5609 - Add support for (add/remove/update) of acceptors via config reload
  • ARTEMIS-5625 - Support AMQP address federation links using filters from the address bindings

  • ARTEMIS-5656 - Prevent bi-directional AMQP Address Federation from creating unneeded links
  • ARTEMIS-5659 - Support masking JDBC URL
  • ARTEMIS-5669 - Send Modified(delivery-failed) to AMQP bridge receivers by default when the target address is full
  • ARTEMIS-5682 - brokerProperties: support clearing of collections and maps with the removeValue -

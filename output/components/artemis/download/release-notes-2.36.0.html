ActiveMQ Artemis 2.36.0 Release Notes

  • A list of commits.

  • Please see the “Versions” chapter in the User Guide for upgrade instructions.

Note: This release requires use of Java 11 or above.

Bugs Fixed

  • ARTEMIS-2892 - Replication stopped working after many failovers
  • ARTEMIS-3509 - STOMP Websocket client disconnected after ConcurrentModificationException in Broker
  • ARTEMIS-4667 - JMX Recovery operation triggers ClassCastException
  • ARTEMIS-4747 - UNREACHABLE_CODE.EXCEPTION This statement in the source code is unreachable
  • ARTEMIS-4794 - CoreBridge: Duplicate message when bridge is stopped/Lost message when bridge is paused while messages being produced to target node.
  • ARTEMIS-4816 - Docker image echo’s admin credentials during startup
  • ARTEMIS-4817 - AMQP Federation address policy doesn’t allow credit configuration to override parent settings
  • ARTEMIS-4820 - AMQP TTL message property being treated as int instead of uint
  • ARTEMIS-4821 - OperationContext may leak between servers in embedded/test mode on SendAckThread on a bridge
  • ARTEMIS-4826 - ICAST_INTEGER_MULTIPLY_CAST_TO_LONG value too high
  • ARTEMIS-4827 - UNREACHABLE_CODE
  • ARTEMIS-4902 - Downstream federation example not compliant with xsd
  • ARTEMIS-4905 - PeriodicConnectionExpiry plugin comparison error treating seconds as milli seconds
  • ARTEMIS-4910 - DivertConfiguration encoding is broken
  • ARTEMIS-4912 - Filter is not enforced when sending message to queue via web console
  • ARTEMIS-4914 - Clarify docs on core + client ID
  • ARTEMIS-4919 - AMQP protocol level errors not handled cleanly
  • ARTEMIS-4922 - Spurious warn message when OpenWire client disconnects
  • ARTEMIS-4925 - Mirroring and Replication do not work in combination
  • ARTEMIS-4931 - BridgeConfiguration encoding is broken
  • ARTEMIS-4936 - Verify response correlationId in Core client
  • ARTEMIS-4941 - Remove update estimate after ARTEMIS-3067
  • ARTEMIS-4948 - Clarify failover-on-shutdown-doc

  • ARTEMIS-4950 - MBeanGuard throws an NPE
  • ARTEMIS-4951 - Multiple connection router occurrences with same policy cause parser error

Apache ActiveMQ® is a popular and powerful open source messaging and [Integration Patterns](documentation/enterprise-integration-patterns) server. Released under the [Apache 2.0 License](http://www.apache.org/licenses/LICENSE-2.0).

Grab yourself a [download](/download/), review the [Getting Started Guide](documentation/getting-started), surf the [FAQ](documentation/faq), or start [contributing](/contributing/).

---

<div class="feature-grid">

<div class="feature-card">
  <h4>Multi-Protocol</h4>
  <p>Supports <a href="documentation/openwire">OpenWire</a>, <a href="documentation/stomp">STOMP</a>, <a href="documentation/amqp">AMQP</a> 1.0, and <a href="documentation/mqtt">MQTT</a> 3.1 — connect from Java, C, C++, Python, Ruby, .NET and more.</p>
</div>

<div class="feature-card">
  <h4>Enterprise Integration</h4>
  <p>Full support for <a href="documentation/enterprise-integration-patterns">Enterprise Integration Patterns</a> in both the JMS client and the broker.</p>
</div>

<div class="feature-card">
  <h4>JMS & Jakarta Messaging</h4>
  <p>Full JMS 1.1 and J2EE 1.4+ support, with <a href="documentation/jms2">partial Jakarta Messaging 3.1 & JMS 2.0</a> support including transient, persistent, transactional and XA messaging.</p>
</div>

<div class="feature-card">
  <h4>High Availability</h4>
  <p>Designed for clustering, client-server and peer communication. Fast <a href="documentation/persistence">persistence</a> via JDBC and a high-performance journal.</p>
</div>

<div class="feature-card">
  <h4>Spring & J2EE Ready</h4>
  <p><a href="documentation/spring-support">Spring integration</a> out of the box. Tested with TomEE, Geronimo, JBoss, GlassFish and WebLogic. Includes <a href="documentation/resource-adapter">JCA 1.5 resource adaptors</a>.</p>
</div>

<div class="feature-card">
  <h4>Flexible Transport</h4>
  <p>Pluggable <a href="documentation/uri-protocols">transport protocols</a>: TCP, SSL, NIO, UDP, multicast, JGroups, JXTA and <a href="documentation/how-do-i-use-activemq-classic-using-in-jvm-messaging">in-VM</a>.</p>
</div>

<div class="feature-card">
  <h4>Web & REST</h4>
  <p><a href="documentation/rest">REST</a> API for language-neutral messaging. <a href="documentation/ajax">Ajax</a> support for web streaming via pure DHTML.</p>
</div>

<div class="feature-card">
  <h4>Advanced Features</h4>
  <p><a href="documentation/message-groups">Message Groups</a>, <a href="documentation/virtual-destinations">Virtual Destinations</a>, <a href="documentation/wildcards">Wildcards</a>, <a href="documentation/composite-destinations">Composite Destinations</a>, and <a href="documentation/features">much more</a>.</p>
</div>

</div>
